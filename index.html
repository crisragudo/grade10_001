A. Introduction

(5 Minutes)

This section introduces the core idea of persistent data storage through relatable questions, setting the stage for understanding why File I/O is a fundamental concept in programming.

Teacher's Activity:

Greet students and briefly review the previous lesson on basic programming syntax.

Motivation Questions:

"When you save a document in Google Docs or Microsoft Word, where does it go?"
"How does a video game remember your high score after you close it?"

Defining File I/O:

Using a simple diagram, define **File I/O** as the process of a program interacting with files. Explain that 'I' is for **Input** (reading from a file) and 'O' is for **Output** (writing to a file).

B. Development of the Lesson

(20 Minutes)

1. Explaining "Reading" vs. "Writing" (10 mins)

This part breaks down the two core operations of File I/O using simple analogies and code snippets. The interactive cards encourage active participation to reveal the definitions.

2. Visual and Interactive Activity (10 mins)

This activity simulates the File I/O process in a hands-on, collaborative way. It's designed to be performed in a classroom without computers, solidifying the abstract concepts through role-playing.

Scenario:

"A program needs to keep track of a student's top 5 favorite Filipino foods."

Simulation Steps:

  1. Divide & Role-Play: Form groups. Each group is a "program." One student is the "processor," another is the "file."
  2. Writing (Output): The "processor" gives a list of 5 Filipino foods to the "file" student. The "file" writes this list on paper (the "text file").
  3. Reading (Input): The "processor" asks the "file" to retrieve the list. The "file" student hands the paper back to be read.

C. Discussion and Wrap-up

(10 Minutes)

The final phase of the lesson reinforces the learning through discussion and a quick, individual assessment. It ensures students can articulate the key concepts in their own words.

Why is File I/O important?

It allows data to be persistent. This means data is not lost when the program closes, unlike data stored only in the computer's temporary memory (RAM).

Quiz: File I/O Fundamentals

Test your understanding with these five questions based on the lesson plan. You can reveal the answers at the end.

1. In your own words, define File I/O.

Answer: File I/O is the process of a program interacting with a file by either reading data from it or writing data to it.

2. Imagine a program needs to save a new high score. Is this an example of reading or writing?

Answer: This is an example of **writing**, because the program is sending data (the new high score) to the file.

3. What does the "I" in File I/O stand for?

Answer: The "I" stands for **Input**.

4. What does the "O" in File I/O stand for?

Answer: The "O" stands for **Output**.

5. What is the main benefit of using File I/O?

Answer: The main benefit is that it allows data to be **persistent**, meaning the data is not lost when the program closes.
